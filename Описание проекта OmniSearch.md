# **🔍 Проект OmniSearch: Система семантического поиска по видео**

## **📖 Описание проекта**

**OmniSearch** — это распределенная микросервисная система, предназначенная для глубокого анализа и семантического поиска внутри видеофайлов.

Система позволяет пользователю загрузить видеофайл, после чего под капотом происходит автоматическая декомпозиция медиа: извлечение аудиодорожки и нарезка видеоряда на отдельные кадры. Далее в дело вступает машинное обучение: аудио транскрибируется в текст, а изображения переводятся в векторные представления (эмбеддинги). Это позволяет осуществлять умный поиск по контенту видео (как по смыслу сказанных слов, так и по визуальным объектам в кадре).

## **🏗 Архитектура и Data Flow (Поток данных)**

Проект построен на принципах асинхронной микросервисной архитектуры. Общение между тяжелыми вычислительными узлами происходит через файловую систему (Shared Volumes) и API.

1. **Upload & UI:** Пользователь загружает .mp4 файл через Frontend. Клиент начинает асинхронно опрашивать (polling) бэкенд для отслеживания статуса.  
2. **Orchestration:** Backend (Ktor) принимает файл, сохраняет его, создает запись в БД со статусом PROCESSING\_MEDIA и вызывает системную утилиту (Video Engine).  
3. **Media Decoding:** Video Engine (Go) запускается как подпроцесс, читает видеофайл и параллельно извлекает .wav аудио (через FFmpeg) и нарезает кадры .jpg (через GoCV).  
4. **AI Processing:** По завершении нарезки, Backend делает HTTP-запрос к ML Engine. Python-сервис берет аудио и кадры, прогоняет их через нейросети (Whisper \+ CLIP) и возвращает готовые эмбеддинги и транскрипцию.  
5. **Ready:** Backend сохраняет результаты поиска и меняет статус на READY. Frontend отображает успешное завершение пользователю.

## **🛠 Технологический стек**

Проект разделен на 4 независимые зоны ответственности, каждая из которых использует наиболее подходящие инструменты:

### **1\. Backend (Оркестратор и API)**

* **Язык:** Kotlin / Java  
* **Фреймворк:** Ktor  
* **База данных / Хранилище:** PostgreSQL / MinIO (S3) для хранения медиа.  
* **Особенности:** Управление подпроцессами, корутины для асинхронной работы, DI, маршрутизация HTTP-запросов к ML-сервису.

### **2\. Frontend (Пользовательский интерфейс)**

* **Технологии:** React, TypeScript, Vite, CSS Modules.  
* **Сетевое взаимодействие:** Axios / Fetch API, механизмы Polling.  
* **Функционал:** SPA-приложение, Drag-and-drop загрузка, динамические стейты загрузки.

### **3\. Video Engine (Медиа-движок)**

* **Язык:** Go (Golang)  
* **Библиотеки:** gocv.io/x/gocv (CGO биндинги для C++ библиотеки OpenCV).  
* **Системные зависимости:** FFmpeg, OpenCV 4.x.  
* **Особенности:** Fail-Fast CLI-утилита, параллельная обработка через Goroutines.

### **4\. ML Engine (Движок нейросетей)**

* **Язык:** Python  
* **Фреймворк API:** FastAPI  
* **ML Стек:** PyTorch, векторные хранилища (Vector Store), Whisper, Vision-модели (CLIP).

## **👥 Состав команды и зоны ответственности**

Проект разрабатывается кросс-функциональной командой. Вся кодовая база разделена на независимые директории, за каждую из которых отвечает своя роль:

### **👑 Тимлид (Team Lead)**

**Ответственность:** Техническое руководство проектом, координация работы подкоманд и обеспечение целостности системной архитектуры.

* Принятие ключевых архитектурных решений и проектирование контрактов взаимодействия (API) между микросервисами.  
* Управление процессами разработки (Kanban), приоритизация задач и устранение технологических блокеров.  
* Проведение кросс-сервисного Code Review и контроль соблюдения нефункциональных требований (NFR).  
* Синхронизация работы ML-движка, Backend-оркестратора, Frontend-приложения и системных утилит (Video Engine).

### **💻 Frontend (Клиентская часть)**

**Ответственность:** Разработка пользовательского интерфейса (SPA) и обеспечение бесшовного UX при загрузке тяжелых файлов.

* Разработка компонентов загрузки (Uploader, DragAndDropZone) и строки поиска.  
* Управление состоянием приложения (стейт-менеджмент прогресса загрузки и обработки видео).  
* Сетевое взаимодействие с Backend-API (отправка multipart/form-data, опрос статуса).

### **⚙️ Backend (Оркестрация и хранение)**

**Ответственность:** Разработка центрального узла системы (Ktor API), который связывает все остальные сервисы воедино.

* Маршрутизация запросов (Routing) и настройка безопасности/политик (CORS, обработка исключений).  
* Интеграция с S3-совместимым хранилищем (MinioVideoStorage) для надежного сохранения сырых видео.  
* Работа с базой данных (PostgresVideoRepository) для ведения учета загруженных видео и стейт-машины их обработки.  
* Инициация пайплайнов (вызов Go-утилиты и отправка команд в ML-сервис).

### **🎞️ Video Engine (Медиа-процессинг)**

**Ответственность:** Создание высокопроизводительной утилиты для декодирования видео на системном уровне.

* Написание потокобезопасного (thread-safe) кода на Golang.  
* Интеграция с FFmpeg (audio/extractor.go) для быстрого дампа аудиодорожек.  
* Работа с CGO и OpenCV (video/sampler.go) для захвата и сохранения кадров с жестким контролем утечек памяти (Memory Management).

### **🧠 ML Engine (Машинное обучение)**

**Ответственность:** Разработка AI-ядра для понимания контента видео.

* Создание и поддержка инференс-сервиса на базе FastAPI (inference.py).  
* Интеграция моделей транскрибации (Speech-to-Text) для извлечения текста из аудио.  
* Интеграция Vision-моделей для перевода набора изображений в многомерные векторы (эмбеддинги).  
* Организация и управление векторным хранилищем (vector\_store.py) для быстрого семантического поиска.

### **🐳 DevOps / Infra (Инфраструктура)**

**Ответственность:** Контейнеризация, сборка и развертывание проекта.

* Написание оптимизированных (Multi-stage) Dockerfile для каждого из сервисов. Особый фокус на сборку тяжелых образов (Go \+ OpenCV, Python \+ PyTorch).  
* Описание общей инфраструктуры для локальной разработки в docker-compose.yml (поднятие БД, MinIO, Backend, ML и Frontend в единой сети).  
* Настройка CI/CD пайплайнов (GitHub Actions в .github/workflows/ci-pipeline.yml) для автоматического тестирования и сборки проекта.
