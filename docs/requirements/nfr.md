# Нефункциональные требования - MVP v.0.1


### 1. Производительность для Синхронных операций
* **Поиск (`GET /api/v1/search`):** Время ответа в 95-м перцентиле (p95) < 500 ms. Максимальное допустимое (p99) < 1000 ms. Включает векторизацию запроса, поиск в векторной БД и сбор метаданных из PostgreSQL.
* **Загрузка (`POST /api/v1/upload`):** Время ответа (p95) < 200 ms, (p99) < 500 ms. Оценивается время до выдачи статуса `202 Accepted` и `video_id`(не учитывая фоновую обрабтку).
* **UI Frontend:** Рендеринг и перестроение сетки видеороликов после получения ответа от бэкенда занимает < 100 ms

### 2. Пропускная способность для Асинхронных операций
* **End-to-End Ingestion:** RTF ≤ 1.0. Время от загрузки видео до перехода в статус `READY` не превышает длительность самого видеоролика.
* **Извлечение аудио (VE):** < 5 сек. на 1 минуту видео (использование stream copy через FFmpeg).
* **Сэмплирование кадров (VE):** Скорость обработки > 100 FPS (чтение кадров OpenCV, базовая дедупликация и запись уникальных `.jpg`).
* **Транскрибация аудио (VE):** RTF ≤ 0.5. Модель Whisper (tiny/base) на CPU обрабатывает 1 минуту аудио максимум за 30 сек.

### 3. Ограничения ресурсов
* **Ktor Backend:** Максимум 512 MB RAM, 1 CPU Core. Нагрузка преимущественно in/out.
* **C++ Video Engine:** Максимум 500 MB RAM на процесс, 1 CPU Core. Контроль утечек памяти
* **ML Engine:** Максимум 3 GB RAM, 2-4 CPU Cores. Включаетт загрузку моделей в оперативную память при старте контейнера
* **PostgreSQL:** Максимум 512 MB RAM, 1 CPU Core. Хранение только текстовых метаданных и статусов.
* **Vector DB:** Максимум 1 GB RAM, 1 CPU Core. Хранение HNSW индексов в оперативной памяти для быстрого поиска.

### 4. Ограничения входных данных 
* **Максимальный размер файла:** Ограничения со стороны видеохостинга.
* **Максимальная длительность:** Ограничения со стороны видеохостинга.
* **Поддерживаемые форматы:** Ограничения со стороны видеохостинга.
* **Частота кадров (Сэмплирование):** 1-2 FPS. Без учета оптимизации кадров.
